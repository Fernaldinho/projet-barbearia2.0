# REGRA GLOBAL — MCP STITCH + PROMPT ENGINE + LOOP AUTOMÁTICO

## Objetivo
Garantir que sempre que o MCP Stitch for utilizado, o sistema gere interfaces funcionais, organizadas, responsivas e integradas ao projeto através de criação automática de prompt e loop de validação contínuo.

---

## Regra Principal
Sempre que o MCP Stitch for utilizado, é obrigatório executar:
1. Habilidade de criação de prompt
2. Habilidade de loop automático
3. Validação contínua
4. Ajustes até resultado funcional

**Fluxo obrigatório:**
`PROMPT → STITCH → LOOP → VALIDAÇÃO → AJUSTE → STITCH → LOOP`

**Nunca executar Stitch diretamente.**

---

## Detecção automática
Ativar este workflow quando houver:
* criação de tela
* criação de página
* criação de layout
* criação de UI
* criação de componentes
* atualização visual
* estrutura de interface
* dashboard
* formulários
* landing pages
* painéis administrativos
* páginas responsivas
* qualquer ação envolvendo MCP Stitch

---

## Etapa 1 — Análise
Antes de executar o Stitch, o sistema deve entender:
* objetivo da tela
* funcionalidade principal
* fluxo do usuário
* tipo de sistema
* integração com backend
* responsividade
* regras de negócio
* estados da interface
* UX esperada
* padrão visual do projeto
* componentes necessários
* ações do usuário

---

## Etapa 2 — Criação de Prompt Técnico
Gerar automaticamente um prompt estruturado para o Stitch contendo:
### Objetivo da tela
Descrição clara do que será criado
### Estrutura da interface
Componentes, Seções, Layout, Organização
### Funcionalidades
Botões, Inputs, Ações, Fluxos
### Responsividade
Mobile, Tablet, Desktop
### Integração
API, Banco, Backend, Estado da aplicação
### Comportamento
Loading, Erro, Sucesso, Validação
### UX
Clareza, Organização, Facilidade de uso, Fluxo intuitivo
### Acessibilidade
Contraste, Leitura, Tamanho de botões, Inputs
### Padrão visual
Cores do projeto, Tipografia, Componentização, Design consistente

---

## Etapa 3 — Execução do Stitch
Executar MCP Stitch usando o prompt gerado.
Nunca executar sem prompt estruturado.

---

## Etapa 4 — Loop Automático
Após gerar a interface, iniciar loop de validação.
Validar: layout, componentes, botões, inputs, responsividade, integração, UX, erros visuais/técnicos, padrões e acessibilidade.

---

## Etapa 5 — Correção automática
Se houver problema:
1. Identificar erro
2. Gerar novo prompt
3. Ajustar estrutura
4. Executar Stitch novamente
5. Validar novamente
6. Repetir loop até estar correto

---

## Critério de parada
Parar somente quando:
* interface funcional, layout correto, responsivo, sem erros, integrado, UX adequada, código limpo, pronto para produção.

---

## Regra obrigatória
**Nunca:** usar Stitch direto
**Sempre:** criar prompt → executar Stitch → validar → corrigir → repetir
